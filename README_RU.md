### Введение

FoxCode — это терминальный CLI AI-ассистент для кодирования, написанный на Python. Он поддерживает множество AI-моделей, предоставляя интеллектуальную генерацию кода, операции с файлами, планирование задач, управление рабочими процессами и другие функции, помогая разработчикам повысить эффективность кодирования.

### Возможности

#### Основные функции
- **Поддержка множества моделей**: OpenAI, Anthropic Claude, DeepSeek, Step, локальные модели
- **Операции с файлами**: Чтение, запись, редактирование, поиск файлов
- **Выполнение Shell**: Безопасное выполнение команд в терминале
- **Планирование задач**: Управление выполнением сложных задач
- **Управление рабочими процессами**: Стандартизированный рабочий процесс разработки (Дизайн → Код → Тест → Слияние → Пуш)
- **Система навыков**: Расширяемый фреймворк навыков с поддержкой динамической загрузки
- **Протокол MCP**: Поддержка Model Context Protocol для подключения внешних инструментов
- **Песочница безопасности**: Настраиваемая политика безопасности выполнения команд
- **Интерактивный интерфейс**: Современный TUI-интерфейс
- **Управление сессиями**: Сохранение и восстановление истории диалогов с поддержкой шифрования
- **Несколько режимов работы**: По умолчанию, Build, режим планирования, режим автоматического принятия правок
- **Семантическая индексация кода**: Семантический поиск кода на основе векторных эмбеддингов с поддержкой инкрементальных обновлений и генерации графа зависимостей
- **Управление базой знаний**: Хранение знаний между сессиями с поддержкой категоризации, тегов и семантического поиска
- **Сжатие контекста**: Интеллектуальное сжатие диалогов и дистилляция знаний для оптимизации обработки длинных диалогов
- **Интеллектуальное планирование задач**: Автоматическое разбиение задач, анализ зависимостей и топологическая сортировка
- **Анализ структуры проекта**: Автоматическое определение технологического стека, оценка качества кода и распознавание архитектурных шаблонов
- **Анализ ошибок**: Интеллектуальный разбор ошибок, анализ корневых причин и предложения по исправлению
- **Продвинутая отладка**: Условные точки останова, точки останова в логах, наблюдение за переменными и анализ стека вызовов
- **Анализ производительности**: Анализ времени выполнения, отслеживание памяти и выявление узких мест
- **Сканирование безопасности**: Обнаружение уязвимостей, сканирование конфиденциальной информации и проверка зависимостей
- **Форматирование кода**: Форматирование кода на нескольких языках (Python, JavaScript, TypeScript и др.)
- **Предложения по рефакторингу**: Обнаружение запахов кода, предложения по шаблонам проектирования и планы рефакторинга
- **Разрешение зависимостей**: Анализ зависимостей Python/Node.js и обнаружение конфликтов
- **Генерация тестов**: Генерация тестов в стиле pytest, обработка граничных случаев и поддержка TDD
- **Генерация документации**: Автоматическая генерация документации API, docstring и README
- **Продвинутые Git-операции**: Умные коммиты, разрешение конфликтов и управление ветками
- **Мультимодальная обработка**: Анализ изображений, генерация диаграмм Mermaid/PlantUML

### TUI-интерфейс

FoxCode предоставляет полнофункциональный TUI (Terminal User Interface), построенный на Textual, с воссозданием интерактивного опыта Claude Code 1:1.

#### Горячие клавиши TUI

| Клавиша | Действие |
|---------|----------|
| `Enter` | Отправить сообщение |
| `Shift+Enter` | Новая строка |
| `Ctrl+L` | Очистить экран |
| `Ctrl+N` | Новая сессия |
| `Ctrl+S` | Сохранить сессию |
| `Ctrl+B` | Переключить боковую панель |
| `Ctrl+T` | Цикл режимов (build/plan/accept) |
| `F1 / ?` | Справка |
| `F11` | Полноэкранный режим |
| `PageUp / PageDown` | Прокрутка вверх/вниз |
| `Ctrl+A` | Выделить весь текст |
| `Ctrl+E` | Курсор в конец строки |
| `Ctrl+K` | Удалить до конца строки |
| `Ctrl+W` | Удалить слово |
| `Ctrl+U` | Очистить строку |
| `Esc` | Отмена |
| `Ctrl+C` | Выход только когда фокус на поле ввода; в области чата не выходит (для копирования) |
| `Ctrl+Q` | Выход из FoxCode |
| `Ctrl+Y` | Копировать последний ответ AI (беззвучно, без подсказки) |
| `C` | Копировать выбранное сообщение (сначала щёлкните по сообщению; беззвучно) |
| `D` | Удалить выбранное сообщение |

#### TUI слеш-команды (локальные, не отправляются AI)

Ввод, начинающийся с `/`, выполняется как локальная инструкция, а не отправляется AI-агенту. Команды выполняются немедленно, даже во время потоковой выдачи AI.

**Порядок разрешения команд:**
1. Сначала поиск среди встроенных команд TUI (таблица ниже); если найдено — выполняется внутри TUI.
2. Если TUI не распознаёт команду, она **как есть** передаётся обработчику команд CLI (`cli._handle_command`). Это означает, что длинный список интерактивных команд CLI (`/init` `/features` `/index` `/search` `/kb` `/debug` `/test` `/doc` `/git` `/topic` и т.д.) **не нужно** повторно реализовывать в TUI — просто введите их в TUI.
3. Только если ни TUI, ни CLI не распознали команду, в чате отображается ошибка «неизвестная команда».

| Встроенная команда TUI | Описание |
|------------------------|----------|
| `/help` | Показать список команд TUI |
| `/clear` | Очистить чат (отменяет текущий вывод) |
| `/save` | Сохранить текущую сессию |
| `/mode [name]` | Установить режим: `build` / `plan` / `accept_edits` |
| `/new` | Начать новую сессию (отменяет текущий вывод) |
| `/sidebar` | Переключить боковую панель |
| `/fullscreen` или `/fs` | Полноэкранный режим |
| `/theme [name]` | Сменить тему (dark / light / dark-ansi / light-ansi / dark-daltonized / light-daltonized) |
| `/history` | Показать историю ввода |
| `/quit` или `/exit` | Выход из FoxCode |
| `//text` | Отправить буквальное сообщение, начинающееся с `/`, AI |

> Все остальные `/команды` передаются CLI. Вывод CLI захватывается и отображается в чате (TUI временно перенаправляет `console` CLI, чтобы не нарушать интерфейс терминала).

#### Особенности TUI

- **Многострочный ввод**: TextArea с Shift+Enter для новой строки
- **История команд**: Навигация по истории стрелками вверх/вниз
- **Панель действий с сообщениями**: E=редактировать/C=копировать/D=удалить/J=перейти при выборе сообщения
- **Анимированный индикатор**: Стандартная анимация `-/\\|` с детекцией зависания и эффектом Glimmer
- **Талисман Clawd**: ASCII-лиса с 4 анимациями поз
- **Полноэкранный режим**: F11 для переключения, скрыть боковую панель для расширения основной панели
- **Персистентность сессий**: Автосохранение/восстановление сессий в `~/.foxcode/sessions/`
- **6 тем**: dark / light / dark-ansi / light-ansi / dark-daltonized / light-daltonized
- **Фирменный цвет**: `#ffd56b` тёплый золотой
- **Система диалогов**: Справка, подтверждение, запрос разрешений, просмотр Diff, предупреждение о стоимости

### Рабочий процесс

FoxCode предоставляет стандартизированный рабочий процесс разработки:


Планирование дизайна → Реализация кода → Оценка качества → Локальное тестирование → Слияние в основную ветку → Интеграционное тестирование → Пуш ветки

Каждый этап отслеживается по статусу:
- Ожидание (pending)
- В процессе (in_progress)
- Завершено (completed)
- Ошибка (failed)
- Пропущено (skipped)
- Заблокировано (blocked)

### Функции безопасности

- **Режим песочницы**: Поддержка чёрного/белого списков для ограничения выполнения команд
- **Валидация команд**: Предотвращение опасных инъекций команд
- **Фильтрация конфиденциальной информации**: Автоматическая фильтрация API-ключей и другой конфиденциальной информации
- **Шифрование сессий**: Поддержка шифрованного хранения данных сессий
- **Аудит безопасности**: Запись журналов событий безопасности

### FAQ

**В:** Как включить минималистичный режим терминала, если кажется, что слишком много контента?

**О:** `/topic minimalism`

**В:** Как использовать локальные модели?

**О:**

foxcode --model local --base-url http://localhost:8000/v1

**В:** Как просмотреть использование токенов?

**О:** Используйте команду `/token` в интерактивной сессии.

**В:** Как включить режим отладки?

**О:**

foxcode --debug

**В:** Где находятся файлы конфигурации?

**О:** Конфигурация уровня проекта — `.foxcode.toml` в корне проекта, конфигурация уровня пользователя — `~/.foxcode/config.toml`.
